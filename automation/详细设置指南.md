# 🔧 微信小程序自动化 - 详细设置指南

## 问题：端口连接被拒绝

如果遇到 `端口 20774 连接被拒绝` 的错误，说明自动化端口没有启动。

## 🎯 解决方案（按顺序尝试）

### 方案 1：使用 launch 模式启动

使用 `launch` 模式会自动启动开发者工具和自动化端口。

**步骤：**

1. **关闭微信开发者工具**（完全退出）

2. **修改 `automation/setup.js` 第 120 行左右**，改为使用 launch 模式：

```javascript
// 在 main() 函数中
// 方式1: 启动开发者工具（如果工具未打开）
miniProgram = await launchMiniProgram();  // 取消注释这行

// 方式2: 连接到已打开的开发者工具（推荐，更快）
// miniProgram = await connectToMiniProgram();  // 注释掉这行
```

3. **运行 `run.cmd`**

### 方案 2：手动开启服务端口（推荐）

**步骤：**

1. **打开微信开发者工具**

2. **点击左上角 "工具" 菜单 → 设置 → 安全设置**
   - ✅ 勾选 **服务端口**
   - 查看端口号（默认应该是 9420）

3. **记录端口号**
   - 如果显示的端口不是 20774，需要修改配置文件

4. **查看端口号的方法：**
   - 打开开发者工具后
   - 工具 → 设置 → 安全设置 → 服务端口
   - 勾选后会显示端口号

### 方案 3：使用默认端口 9420

微信开发者工具的默认自动化端口通常是 **9420**，而不是 20774。

**修改配置为默认端口：**

1. 修改 `automation/setup.js` 第 61 行：
```javascript
const miniProgram = await automator.connect({
  wsEndpoint: 'ws://localhost:9420', // 改回默认端口
});
```

2. 修改配置文件中的端口（如果需要）

### 方案 4：检查端口占用

**检查端口是否被其他程序占用：**

打开命令行运行：
```cmd
netstat -ano | findstr 20774
```

如果没有输出，说明端口没有被监听（开发者工具的自动化服务没有启动）。

### 方案 5：完全手动配置

1. **打开微信开发者工具**

2. **加载项目：** 微信Flask-小程序

3. **点击右上角 "详情"**

4. **切换到 "本地设置" 标签**

5. **找到并勾选：**
   - ✅ 不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书
   - ✅ 启用自动化（如果有这个选项）

6. **在 "工具" → "设置" → "安全设置" 中：**
   - ✅ 勾选 "服务端口"
   - 记录显示的端口号

7. **更新 setup.js 中的端口号为实际显示的端口**

## 🔍 端口号说明

微信开发者工具的自动化端口有两种模式：

1. **默认模式（9420）**
   - 工具 → 设置 → 安全设置 → 服务端口
   - 默认端口：9420

2. **项目自定义端口**
   - 在 `project.config.json` 中配置
   - 但并非所有版本的开发者工具都支持

## 📝 建议的配置步骤

1. **使用默认端口 9420**（最稳定）

2. **修改 `setup.js` 使用默认端口：**

```javascript
async function connectToMiniProgram() {
  try {
    console.log('正在连接到已打开的开发者工具...');
    
    const miniProgram = await automator.connect({
      wsEndpoint: 'ws://localhost:9420', // 使用默认端口
    });
    
    // ...
  }
}
```

3. **确保开发者工具中的服务端口已开启**

4. **运行测试**

## ❓ 仍然无法连接？

如果以上方案都不行，请提供以下信息：

1. 微信开发者工具版本号
2. 操作系统版本
3. 是否勾选了"服务端口"
4. 服务端口显示的端口号是多少
5. 运行 `netstat -ano | findstr 9420` 的结果
