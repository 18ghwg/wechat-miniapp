# 微信开发者工具端口说明

## 两种不同的端口

微信开发者工具使用两个不同的端口：

### 1. HTTP 服务端口（例如：55012）

**位置：** 工具 → 设置 → 安全设置 → 服务端口

**用途：** 
- HTTP API 调用
- 命令行调用
- 其他工具集成

**不是用于：** miniprogram-automator 的 WebSocket 连接 ❌

### 2. 自动化 WebSocket 端口（默认：9420）

**位置：** 自动分配，通常是 **9420**

**用途：** 
- miniprogram-automator 的 WebSocket 连接 ✅
- 自动化测试
- 程序控制小程序

**这才是我们需要的端口！**

## 如何启用自动化端口

### 方法一：通过项目配置（推荐）

在 `project.config.json` 中添加：

```json
{
  "automation": {
    "enable": true
  }
}
```

然后**重新打开项目**，自动化端口会自动启用（默认 9420）。

### 方法二：使用 Launch 模式

使用 `automator.launch()` 会自动启用自动化功能，无需手动配置。

## 验证端口是否开启

运行以下命令检查：

```bash
# Windows PowerShell
netstat -ano | findstr 9420

# 如果看到输出，说明端口已开启
```

## 常见问题

### Q: 为什么 55012 连接不上？
A: 因为 55012 是 HTTP 服务端口，不支持 WebSocket 自动化连接。

### Q: 9420 端口在哪里配置？
A: 不需要配置，只要在项目配置中启用 automation，开发者工具会自动在 9420 端口开启 WebSocket 服务。

### Q: 如何确认 9420 端口已开启？
A: 运行 `debug.cmd` 诊断脚本，它会检查端口状态。
