# 任务列表触发器描述优化

## 📋 更新说明

已同步优化任务列表页面的触发器描述，使其与任务编辑页面保持一致，显示更清晰易懂的执行时间描述。

## ✨ 主要改进

### 1. 添加完整的Cron解析逻辑

在任务列表页面（`list/index.js`）添加了与编辑页面相同的cron表达式解析逻辑：

```javascript
generateCronDesc(task)      // 生成cron描述
formatCronValue(value, unit) // 格式化cron值
formatDayOfWeek(value)       // 格式化星期
generateIntervalDesc(task)   // 生成间隔描述
```

### 2. 智能识别并添加周期前缀

现在任务卡片会自动识别配置并添加合适的前缀：

| 配置场景 | 旧显示 | 新显示 |
|---------|--------|--------|
| 每月5号13:00 | `每天 13:00` ❌ | `每月5号 13:00` ✅ |
| 每月14号18:10 | `每天 18:10` ❌ | `每月14号 18:10` ✅ |
| 每周一9:00 | `周一 09:00` ❌ | `每周周一 09:00` ✅ |
| 工作日8:00 | `周一、周二... 08:00` ❌ | `每周周一、周二... 08:00` ✅ |

### 3. 使用口语化表达

- 将"日"改为"号"（符合中文习惯）
- 星期显示为"周一"而非"星期一"
- 固定时间用冒号分隔（HH:MM）

## 🎯 显示效果示例

### 场景1：每月固定日期
```
配置：月份=*, 日期=5, 小时=13, 分钟=0
显示：每月5号 13:00

```
配置：月份=*, 日期=5,15,25, 小时=14, 分钟=0

```

显示：每周周一、周二、周三、周四、周五 08:00
```

### 场景4：特定月份执行
```
配置：月份=1,4,7,10, 日期=1, 小时=9, 分钟=0
显示：在1、4、7、10月的1号 09:00
```

### 场景5：每天固定时间
```
配置：月份=*, 日期=*, 星期=*, 小时=15, 分钟=0
显示：每天 15:00
```

### 场景6：间隔执行
```
配置：trigger_type=interval, interval_minutes=30
显示：每30分钟
```

## 🔧 技术实现

### 1. 任务数据处理

在加载任务列表时，为每个任务动态生成 `trigger_desc`：

```javascript
const processedTasks = response.data.map(task => {
  const triggerDesc = this.generateTriggerDesc(task);
  return Object.assign({}, task, {
    is_enabled: Boolean(task.is_enabled),
    trigger_desc: triggerDesc  // 添加触发器描述
  });
});
```

### 2. 支持多种触发器类型

- **Cron触发器**：完整解析月、日、星期、时、分、秒
- **Interval触发器**：显示间隔周期（周、天、小时、分钟、秒）
- **Date触发器**：显示"单次执行"

### 3. 测试模式兼容

测试模式和正式模式都使用相同的描述生成逻辑，确保一致性。
