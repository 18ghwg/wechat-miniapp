# Cron执行时间描述示例

## 📋 优化说明

优化了执行时间的中文描述，使其更加清晰易懂，符合中文表达习惯。

### 主要改进点：
1. ✅ 将"日"改为"号"（如：5日 → 5号）
2. ✅ 添加"每月"、"每周"等明确的周期前缀
3. ✅ 智能识别不同场景，生成最自然的描述

---

## 💡 设计原则

### 1. 明确周期
- 月份为`*`且指定日期 → 添加"每月"
- 月份和日期都为`*`且指定星期 → 添加"每周"
- 所有都为`*` → 显示"每天"、"每小时"、"每分钟"等

### 2. 自然表达
- 使用"号"而不是"日"（符合口语习惯）
- 使用"周一"而不是"星期一"（更简洁）
- 固定时间用冒号分隔（如：13:00）

### 3. 避免歧义
- 明确周期范围（每月/每周/每天）
- 时间格式统一（HH:MM）
- 列表用顿号分隔（1、2、3）

### 4. 简洁清晰
- 去除冗余词汇
- 保持描述简短
- 重点突出执行时间


### 优化后
```
每月5号 13:00
✅ 明确是每月执行
✅ "号"更符合中文习惯
✅ 一眼就能看懂执行规则
```

## 📝 注意事项

1. **日期和星期同时指定**：cron中这两个字段是OR关系，会在满足任一条件时执行
2. **跨年月日期**：2月只有28/29天，设置31号时2月不会执行
3. **时区问题**：所有时间基于服务器时区（Asia/Shanghai）
