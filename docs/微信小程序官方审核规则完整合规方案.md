##### 3.4.1 数据收集使用规范（核心规则）

**官方要求**：
```
在收集和使用用户任何数据时：
1. 必须明确告知用户该数据的用途
2. 确保经过用户明确同意和授权
3. 应在用户同意和授权的范围内进行合理使用

```

**适用数据类型**：
- 地理位置
- 用户通讯录
- 用户手机号码

- 用电数据
- 考勤记录

**必须修复的问题**：

1. **登录页面** - 需添加隐私政策同意
   ```javascript
   // ❌ 当前代码（pages/login/index.js）
   onWechatLogin() {
     wx.login({ ... });  // 直接登录，未告知隐私政策
   }
   
   // ✅ 应该改为
   onWechatLogin() {
     if (!this.data.agreed) {
       wx.showModal({
         title: '提示',

       });
       return;
     }
     wx.login({ ... });
   }
   ```

2. **绑定页面** - 需添加数据使用说明
   ```xml
   <!-- ❌ 当前缺失 -->
   
   <!-- ✅ 应该添加 -->
   <view class="privacy-notice">
     <text>您的账号信息将仅用于电费查询服务，查看《隐私政策》</text>
   ```

##### 3.4.8 手机号码授权登录规范（⭐ 重要补充）

根据微信官方文档[《关于收集用户手机号码授权登录案例解析和整改建议》](https://developers.weixin.qq.com/community/develop/doc/0000ccf58c05402478534e85f61809)，平台对小程序内的账号登录功能有严格规范。

**四种常见违规行为**：

| 违规类型 | 说明 | 本小程序检查 |
|---------|------|-------------|
| ⭐⭐⭐⭐⭐ 一进入即要求登录 | 用户未浏览体验任何功能，即要求授权登录 | ⚠️ **需检查** |
| ⭐⭐⭐⭐ 浏览内容需登录 | 查看商品/内容详情需要先登录 | ✅ 通过：可浏览内容 |
| ⭐⭐⭐⭐⭐ 强制登录无法取消 | 没有取消/拒绝按钮，强制授权 | ⚠️ **需检查** |
| ⭐⭐⭐ 内部使用无说明 | 仅限内部人群，首页没有说明 | ⚠️ **需完善** |

**官方整改要求**：

```
✅ 必须遵守的规则：

1. 用户进入小程序，应该提供用户体验浏览小程序内容功能
   - 不得在未体验任何功能页面时，即弹窗要求用户授权手机号码登录
   - 用户应该可以先浏览功能，再自主选择是否登录

2. 登录必须提供取消/拒绝选项
   - 不得强制要求用户登录无法返回
   - 用户可以选择"取消登录"继续浏览

3. 仅限内部特定人群使用的小程序
   - 应在首页页面增加账号说明
   - 进行内部账号鉴权
   - 用户了解页面功能再自主选择授权登录
```

#### 修改7：添加隐私政策更新提醒机制

**新建文件**：`utils/privacy-check.js`

```javascript
/**
 * 隐私政策检查工具
 * 用于检查用户是否已同意最新版本的隐私政策
 */

// 当前隐私政策版本
const CURRENT_PRIVACY_VERSION = '1.0';
const PRIVACY_VERSION_KEY = 'privacy_policy_version';
const PRIVACY_AGREED_KEY = 'privacy_policy_agreed';

/**
 * 检查是否需要显示隐私政策
 */
function checkPrivacyAgreement() {
  try {
    const agreedVersion = wx.getStorageSync(PRIVACY_VERSION_KEY);
    const isAgreed = wx.getStorageSync(PRIVACY_AGREED_KEY);
    
    // 如果未同意或版本不匹配，需要重新同意
    if (!isAgreed || agreedVersion !== CURRENT_PRIVACY_VERSION) {
      return false;
    }
    
    return true;
  } catch (error) {
    console.error('检查隐私政策同意状态失败:', error);
    return false;
  }
}

/**
 * 记录用户已同意隐私政策
 */
function recordPrivacyAgreement() {
  try {
    wx.setStorageSync(PRIVACY_VERSION_KEY, CURRENT_PRIVACY_VERSION);
    wx.setStorageSync(PRIVACY_AGREED_KEY, true);
    wx.setStorageSync('privacy_agreed_time', new Date().toISOString());
    console.log('隐私政策同意状态已记录');
    return true;
  } catch (error) {
    console.error('记录隐私政策同意状态失败:', error);
    return false;
  }
}

/**
 * 清除隐私政策同意状态（用户注销时）
 */
function clearPrivacyAgreement() {
  try {
    wx.removeStorageSync(PRIVACY_VERSION_KEY);
    wx.removeStorageSync(PRIVACY_AGREED_KEY);
    wx.removeStorageSync('privacy_agreed_time');
    console.log('隐私政策同意状态已清除');
  } catch (error) {
    console.error('清除隐私政策同意状态失败:', error);
  }
}

module.exports = {
  checkPrivacyAgreement,
  recordPrivacyAgreement,
  clearPrivacyAgreement,
  CURRENT_PRIVACY_VERSION
};
```

## 总结

### 🎯 核心要点

1. **隐私合规是重中之重**
   - 必须有完整的隐私政策和用户协议
   - 数据收集必须明确告知和获得授权
   - 用户必须有查看和删除数据的权利

2. **测试账号必须提供**
   - 管理员和普通用户都要有
   - 测试账号必须能体验所有功能
   - 提供清晰的使用说明

3. **功能必须完整可用**
   - 无崩溃、无严重Bug
   - 所有按钮都有响应
   - 用户体验流畅

4. ⭐ **登录规范必须遵守（新增重点）**
   - 不得一进入就强制要求登录
