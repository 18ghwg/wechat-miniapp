## 关键发现

### 代码逻辑分析

通过详细的代码分析，发现**当前代码的执行顺序是正确的**：

```javascript
onShow() {
  // 1. _needRefreshCalendar 检查（第192-204行）
  if (this._needRefreshCalendar) {
    // 立即刷新日历
    this.loadCalendarAttendance(calendarYear, calendarMonth);
    this.loadTodayAttendance();
    return; // ⭐ 直接返回，跳过30秒时间检查
  }
  
  // 2. 智能刷新机制（第207-227行）
  if (timeSinceLastRefresh < 30000) {
    // 跳过频繁刷新
    return;
  }
  
  // 3. 正常刷新逻辑
  this.refreshPageData();
}
```

**关键点**:
- ✅ `_needRefreshCalendar` 检查在智能刷新机制**之前**执行
- ✅ 检查通过后直接 `return`，完全跳过30秒时间检查
- ✅ 理论上，从编辑页面返回时应该立即刷新

### 可能的根本原因

既然代码逻辑是正确的，bug可能出在以下几个地方：

1. **标记管理问题**:
   - `_needRefreshCalendar` 标记未被正确设置
   - 标记被提前重置（`onShow()` 多次调用）

2. **异步操作问题**:
   - `loadCalendarAttendance()` 是异步的，可能存在时序问题
   - API 调用失败或超时
   - 回调执行时页面状态已改变

3. **视图更新问题**:
   - `setData()` 调用后视图未及时渲染
   - `updateCalendarDisplay()` 未被正确调用
   - 数据更新但视图未刷新

4. **页面生命周期问题**:
   - 微信小程序的生命周期影响了执行流程
   - `onShow()` 在页面完全显示前被调用

5. **数据加载问题**:
   - API 返回的数据格式不正确
   - `attendanceMap` 未正确更新
   - 日历数据与实际数据不同步
