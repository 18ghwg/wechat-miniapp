# Bug条件探索测试 - 手动测试指南

## 测试目标
验证考勤日历编辑后状态更新Bug的存在，记录反例以证明bug存在。

**重要**: 此测试必须在未修复的代码上执行 - 测试失败确认bug存在

## Bug描述
当用户从编辑页面返回日历页面时，如果距上次刷新时间 < 30秒，即使 `_needRefreshCalendar` 标记为 true，日历显示状态也不会更新。

## 根本原因分析
在 `微信Flask-小程序/pages/attendance/index.js` 的 `onShow()` 函数中：
- `_needRefreshCalendar` 标记检查位于第192-204行
- 智能刷新机制的时间检查（30秒间隔）位于第207-227行
- 当从编辑页面返回时，虽然 `_needRefreshCalendar` 为 true，但如果距上次刷新 < 30秒，刷新逻辑可能被拦截

## 测试前准备
1. 确保微信开发者工具已打开
2. 确保小程序处于测试模式或正常模式
3. 打开考勤日历页面 (`pages/attendance/index`)
4. 打开开发者工具的控制台，观察日志输出

- 但日历数据实际未更新

- "[性能优化] 跳过频繁刷新，距上次刷新: 25 秒"

## 测试场景4: 未打卡到已打卡测试

### 测试步骤
1. 打开考勤日历页面
2. 找到一个显示感叹号（❗）的日期（漏打卡日期）
3. 如果没有漏打卡日期，选择一个未来的日期
4. 点击该日期进入编辑页面

7. **立即**点击返回按钮（在10秒内）

### 验证条件
- 初始状态：日期显示感叹号（`isMissed: true`）或无考勤记录

- 距上次刷新 < 30秒

❌ **测试失败** - 感叹号仍然存在，日历未更新：
- 该日期仍显示感叹号（❗）
- 或者该日期仍显示为空白（无考勤记录）
- 需要手动刷新（下拉刷新）才能看到更新
